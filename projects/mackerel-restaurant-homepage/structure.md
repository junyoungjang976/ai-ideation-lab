# Structure

## 1) 시스템 개요
- 입력(Input): 사용자의 브라우저 접속
- 처리(Process): 정적 HTML/CSS/JS 렌더링
- 출력(Output): 레스토랑 정보가 담긴 반응형 웹페이지

## 2) 구성요소 (Components)
- C1: 히어로 섹션 - 레스토랑 이름, 슬로건, 대표 비주얼
- C2: 메뉴 섹션 - 대표 메뉴와 가격 정보
- C3: 가족 특화 섹션 - 어린이 메뉴, 좌석, 편의시설 안내
- C4: 매장 정보 섹션 - 위치, 영업시간, 연락처
- C5: 네비게이션 - 각 섹션으로 부드럽게 스크롤

## 3) 데이터 흐름 (Flow)
1. 사용자가 홈페이지 접속
2. 네비게이션 또는 스크롤로 원하는 정보 탐색
3. 전화 걸기 / 지도 보기 등 행동 전환

## 4) 인터페이스 (API / UI / Integration)
- 외부 연동: 없음 (정적 페이지)
- 내부 인터페이스: 단일 HTML 파일 내 CSS/JS 인라인

## 5) 운영 관점 (Operations)
- 모니터링/알림: 불필요 (정적 호스팅)
- 장애 대응: 정적 파일이므로 장애 가능성 극히 낮음
- 유지보수 포인트: 메뉴/가격 변경 시 HTML 텍스트 직접 수정
