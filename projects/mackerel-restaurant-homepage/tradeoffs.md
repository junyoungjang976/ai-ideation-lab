# Tradeoffs

## 1) 선택지 목록
- Option A: 단일 HTML 파일 (인라인 CSS/JS)
- Option B: React/Vue SPA
- Option C: WordPress 등 CMS 기반

## 2) 비교 기준
- 비용: A < C < B
- 속도: A(즉시) > B(빌드 필요) > C(서버 필요)
- 유지보수: C(GUI 편집) > A(텍스트 편집) > B(코드 수정)
- 확장성: B > C > A
- 리스크: A(최소) < C(보안 업데이트) < B(의존성 관리)

## 3) 비교 표 (간단히)
- A: 장점 - 즉시 배포, 무료 호스팅, 빠른 로딩 / 단점 - 기능 확장 제한
- B: 장점 - 높은 확장성, 컴포넌트 재사용 / 단점 - 과도한 엔지니어링
- C: 장점 - 비개발자 관리 용이 / 단점 - 서버 비용, 보안 관리

## 4) 결론 (현재 결정)
- 선택: Option A (단일 HTML 파일)
- 선택 이유: 소규모 식당 홈페이지에 프레임워크나 CMS는 과도함. 정적 파일로 충분하고 유지비용 제로.
- 보류/리스크: 메뉴가 자주 바뀌면 CMS 전환 검토
- 나중에 재검토할 조건: 온라인 주문 기능 필요 시 프레임워크 도입 고려
