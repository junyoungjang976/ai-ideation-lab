# Structure

## 1) 시스템 개요
- 입력(Input): 면적/평면 형태, 업종, 좌석 수, 생산량, 동선 제약, 규제/위생 기준, 선호 설비.
- 처리(Process): 데이터 정제 → 배치도/설비 특성 추출 → 모델 학습 → 시뮬레이션 생성 → 제약 검증/스코어링.
- 출력(Output): 2D 배치도(도면/그래프), 설비 리스트, 유사도 점수/제약 만족 리포트.

## 2) 구성요소 (Components)
- C1: 데이터 파이프라인(배치도/설비 리스트 정규화, 라벨링, 피처 생성).
- C2: 패턴 추출 레이어(공간 그래프/동선 패턴, 설비 공존/인접 규칙).
- C3: 생성 모델 레이어(조건부 생성: 배치도 + 설비 리스트 동시 생성).
- C4: 제약 검증 레이어(법규/위생/동선 규칙 위반 체크, 자동 수정 제안).
- C5: 평가/리포팅 레이어(유사도, 제약 만족, 설명 가능성 제공).

## 3) 데이터 흐름 (Flow)
1. 400개 사례를 표준 포맷으로 변환하고 주요 속성을 라벨링한다.
2. 배치도는 공간/장비 그래프와 기하 특성으로, 설비 리스트는 테이블 피처로 변환한다.
3. 조건부 생성 모델을 학습해 배치도와 설비 리스트를 동시 생성한다.
4. 생성 결과를 제약 검증 레이어에서 체크하고 보정한다.
5. 유사도/제약 스코어를 계산해 사용자에게 제공한다.

## 4) 인터페이스 (API / UI / Integration)
- 입력 인터페이스: 프로젝트 기본정보(업종, 면적, 좌석 수, 주방 동선 제약) 입력 폼.
- 출력 인터페이스: 배치도 뷰어(2D), 설비 리스트 테이블, 유사도/제약 리포트.
- 연동: CAD/도면 포맷(예: DXF/JSON) 내보내기, 설비 DB 연동.

## 5) 운영 관점 (Operations)
- 모니터링: 데이터 편향, 제약 위반율, 유사도 점수 추이를 지속 관찰.
- 장애 대응: 입력 누락/오류 시 최소 요구 조건 안내 및 대체 입력값 추천.
- 유지보수 포인트: 업종별 규정 업데이트, 신규 사례 데이터 주기적 재학습.
