# Tradeoffs

## 1) 선택지 목록
- Option A: 규칙 기반(도메인 룰 + 제약 최적화).
- Option B: 데이터 기반 생성 모델(그래프/이미지 생성 중심).
- Option C: 하이브리드(규칙 기반 + 생성 모델 + 제약 검증).

## 3) 비교 표 (간단히)
- A: 장점(제약 준수/설명 용이) / 단점(패턴 다양성 부족, 규칙 유지 비용)
- B: 장점(패턴 다양성/학습 자동화) / 단점(제약 위반 가능, 데이터 부족에 취약)
- C: 장점(정확도와 제약 준수의 균형) / 단점(구현 복잡도 상승)

## 4) 결론 (현재 결정)
- 선택: Option C
- 선택 이유: 400개 데이터의 학습 신호를 활용하되 규정/동선 제약을 강제해 90% 이상 일치율 목표를 현실화하기 위해서.
- 보류/리스크: 하이브리드 파이프라인은 모델·룰의 충돌 해결 로직이 필요.
- 나중에 재검토할 조건: 데이터 규모가 크게 증가하거나 제약 규정이 단순화될 때.
