# Problem

## 1) 한 문장 요약
- 주방 설계 배치도/설비 리스트 400개를 학습해 패턴을 추출하고, AI 시뮬레이션으로 배치도와 리스트를 생성해 실제 데이터와 90% 이상 일치하는 예측 정확도를 달성하는 시뮬레이터 기획.

## 2) 배경과 맥락
- 현재 상황: 주방 설계는 경험 기반의 설계 노하우가 크고, 배치도와 설비 리스트가 동시에 최적화되기 어렵다.
- 왜 지금인가: 400개 규모의 실제 배치도/설비 리스트가 확보되어 데이터 기반 패턴 학습과 자동 생성이 가능해졌다.
- 핵심 문제: 배치도(공간 배치)와 설비 리스트(기기 사양/수량)를 동시에 예측·생성할 수 있는 일관된 시뮬레이션 체계가 부족하다.

## 3) 목표 (Goals)
- G1: 400개 실제 사례에서 공간 배치 패턴(동선, 존 배치, 장비 관계)을 추출한다.
- G2: 입력 조건(면적, 업종, 좌석 수, 동선 제약 등)을 기반으로 배치도와 설비 리스트를 생성한다.
- G3: 실제 데이터와의 일치 확률(유사도/정확도 지표) 90% 이상을 달성한다.
- G4: 설계 결과를 사람이 검토할 수 있도록 생성 근거와 제약 만족 여부를 표시한다.

## 4) 비목표 (Non-goals)
- NG1: 건축 인허가 도면 수준의 상세 설계 자동화는 범위에 포함하지 않는다.
- NG2: 특정 설비 브랜드/모델의 상업적 추천 최적화는 포함하지 않는다.
- NG3: 3D 렌더링 품질 개선은 본 기획의 핵심 목표가 아니다.

## 5) 성공 기준 (Success Metrics)
- 정량: 실제 배치도/리스트와의 유사도 90% 이상(정의된 지표 기준), 제약 위반율 5% 이하.
- 정성: 디자이너가 “초안으로 바로 검토 가능한 수준”이라 평가.

## 6) 제약 조건 (Constraints)
- 데이터: 400개 사례는 데이터 편향 가능성이 있으며, 업종/면적 분포가 불균등할 수 있다.
- 모델: 공간 배치(그래프/기하)와 설비 리스트(테이블)를 동시에 다뤄야 한다.
- 운영: 설계 기준(법규, 위생, 동선 규칙)은 지역/업종별로 상이할 수 있다.
